# WorkoutTracker 🏋️‍♂️📱

WorkoutTracker — Android-приложение на **Kotlin + Jetpack Compose** для ведения тренировок, питания и прогресса.
Текущая реализация хранит данные локально (персонифицированно по пользователю) и объединяет разделы в едином интерфейсе с нижней навигацией.

---

## 🚀 Что есть в приложении сейчас

- **Тренировки**: сессии, упражнения, фото, редактирование каталога упражнений.
- **Питание**: дневник блюд с ингредиентами и КБЖУ, нормы, генерация плана питания.
- **Аналитика**: шаги, погода, КБЖУ за день, история веса, топ упражнений.
- **Статьи**: баланс баллов, покупка статей, предпросмотр/полный доступ.
- **Достижения**: прогресс по достижениям и визуальные статусы.
- **Профиль**: личные данные, аватар, смена темы, выход из аккаунта.

## 🏋️ Раздел «Тренировки»

### Возможности

- Создание тренировочной сессии с датой и списком упражнений.
- Редактирование и удаление тренировок.
- Добавление/удаление упражнений внутри сессии.
- Для упражнения задаются:
  - название;
  - подходы;
  - повторения;
  - вес;
  - фото (опционально).

### Каталог упражнений

- Отдельный диалог «Упражнения».
- Добавление нового упражнения в каталог.
- Редактирование названия и фото упражнения.
- Удаление упражнения из каталога.
- Предзаполненный базовый каталог при первом запуске.

### Валидация

- Название упражнения не пустое.
- Подходы: `1..10`.
- Повторения: `1..100`.
- Вес: `0..1000 кг`.
- Сохранение тренировки доступно только при валидных данных.

## 🍽️ Раздел «Питание»

### Дневник питания

- Добавление, редактирование, удаление записей.
- Группировка записей по датам и типам приёма пищи:
  - завтрак;
  - обед;
  - ужин;
  - перекус;
  - другое.

### Модель блюда

Запись питания хранится как **блюдо из ингредиентов**, а не только как готовые КБЖУ:

- у ингредиента есть КБЖУ на 100 г;
- задаётся вес ингредиента в блюде;
- для съеденной порции задаётся вес;
- итоговые КБЖУ порции считаются автоматически.

### Нормы КБЖУ

- Пользователь задаёт свою дневную норму: калории, белки, жиры, углеводы.
- Есть рекомендованные значения на основе профиля питания.
- Экран «Сегодня» показывает прогресс по нормам.

### План питания

- Генерация плана на текущий день.
- Генерация плана на основе продуктов из «холодильника».
- Замена отдельного приёма пищи в плане.
- Повторное использование вчерашнего плана.
- Сброс текущего плана.

### Валидация

- Контроль диапазонов для порции и макросов.
- Проверка обязательных полей (название блюда, ингредиенты).
- Ограничение по нереалистично большой калорийности порции.

## 📊 Раздел «Аналитика»

### Шаги

- Учёт шагов через `TYPE_STEP_COUNTER` + foreground service.
- Запрос разрешения `ACTIVITY_RECOGNITION`.
- Карточка прогресса: текущие шаги / цель.
- История шагов и ручная корректировка.

### Уведомления о цели шагов

- Включаемый/выключаемый флаг уведомлений.
- Для Android 13+ учитывается `POST_NOTIFICATIONS`.
- Уведомление отправляется при достижении цели не чаще 1 раза в день для текущей цели.

### Погода

- Погода по городу (OpenWeather API).
- Кэширование результата в `SharedPreferences`.
- При проблемах сети используются последние сохранённые данные.
- Ручное обновление погоды.

### КБЖУ и история питания

- Отображение итогов КБЖУ за сегодня.
- История дневных итогов питания.

### Вес

- Сохранение веса с валидацией диапазона.
- История веса (до 60 записей).
- Линейный график прогресса веса.

### Топ упражнений

- Анализ тренировок и упражнений.
- Ранжирование по метрике `вес × повторения × подходы`.
- Количество позиций в топе настраивается в аналитике.

### Настройки аналитики

- Цель по шагам (`1000..50000`).
- Город для погоды + автоопределение по геолокации.
- Вкл/выкл уведомлений.
- Лимит количества упражнений в топе (`1..10`).

## ✅ Как протестировать сканирование штрихкодов

1. Откройте раздел **Питание** и нажмите **+** для добавления приёма пищи.
2. Нажмите кнопку **«Сканировать штрихкод»**.
3. При первом запуске дайте приложению разрешение **CAMERA**.
4. Наведите камеру на EAN/UPC-штрихкод продукта.
5. Дождитесь карточки найденного продукта (название + КБЖУ на 100 г).
6. Проверьте автозаполнение и нажмите:
   - **«Добавить в дневник»** — создаётся запись в дневнике питания;
   - **«Добавить в холодильник»** — продукт добавляется в список холодильника.

### QA-кейсы

- **Онлайн-кейс**: при наличии сети продукт подтягивается из Open Food Facts, затем сохраняется в локальный кэш `product_cache`.
- **Оффлайн-кейс (с кэшем)**: включите авиарежим, повторно отсканируйте уже найденный продукт — карточка должна открыться из кэша.
- **Оффлайн-кейс (без кэша)**: включите авиарежим и сканируйте новый код — должно появиться сообщение о том, что продукт не найден в кэше и сети.

## 🔄 Этап 2: переработка раздела «Питание» (было → стало)

### Было
- Дневник питания и план были на одном длинном экране.
- Холодильник существовал только как диалог для генерации плана.
- Быстрый ввод и сканирование отсутствовали.
- Аналитика питания показывалась полноэкранно с расширенным набором блоков.

### Стало
- Подготовлена Room-основа для разделения сценариев:
  - `nutrition_entries` — дневник приёмов пищи (связка по `userId`);
  - `fridge_items` — продукты холодильника с количеством;
  - `product_cache` — кэш продуктов по barcode.
- Добавлен слой быстрого ввода и сканирования:
  - интеграция ML Kit (barcode/QR);
  - подключение Open Food Facts API;
  - кэширование карточек продукта локально в Room (`product_cache`);
  - runtime-запрос CAMERA + CameraX preview/analyzer в UI диалоге добавления питания;
  - автозаполнение формы и быстрые действия «Добавить в дневник/холодильник».
- Добавлен ViewModel-flow для сценариев:
  - скан → поиск продукта → автозаполнение КБЖУ → запись в дневник;
  - скан → автозаполнение → добавление в холодильник;
  - списание из холодильника (уменьшение количества или удаление).

### Внешние API и кэширование
- **Open Food Facts** (`https://world.openfoodfacts.org/api/v2/product/{barcode}.json`) — используется для получения названия и КБЖУ на 100 г.
- **Стратегия кэша**:
  1. Сначала читаем `product_cache` по `barcode`.
  2. Если данных нет — идём в Open Food Facts.
  3. Успешный ответ сохраняем в `product_cache` и используем для автозаполнения.

### UX flow: скан → автозаполнение → запись в дневник
1. Пользователь нажимает «Быстро добавить».
2. Сканер (ML Kit) распознаёт barcode/QR.
3. По коду запрашивается продукт в Open Food Facts (или из Room-кэша).
4. Открывается карточка продукта: название + КБЖУ на 100 г.
5. Пользователь выбирает граммы/порцию.
6. КБЖУ порции считаются автоматически.
7. Запись сохраняется в `nutrition_entries` (Room, привязка к `userId`).

### UX flow: добавить в холодильник → списать
1. Пользователь открывает интерфейс «Холодильник».
2. Быстрое добавление через скан: код → карточка продукта → количество/единица.
3. Продукт сохраняется в `fridge_items` (Room, привязка к `userId`).
4. При использовании продукта — действие «Списать».
5. Количество уменьшается; если остаток 0 — запись удаляется.

### Аналитика питания
- В Этапе 2 фокус на сводках дня/недели.
- Расширенные аналитические блоки рекомендуется вынести под отдельную иконку в шапке или скрывать настройкой.
